# PERT Doc

## PERT-диаграмма проекта AdventureMaker

## 1. Основные этапы (узлы и работы)

Обозначим задачи (A, B, C...) и зависимости.

### Подготовка и база

- **A** — Анализ ТЗ и проектирование архитектуры
- **B** — Настройка проекта (Android, зависимости, Koin, Retrofit)
- Зависимость: `A -> B`

### Авторизация и пользователь

- **C** — Интеграция Firebase Authentication
- **D** — Получение пользователя из Firebase DB
- **E** — Экран авторизации (UI + ViewModel)
- Зависимости: `C -> D -> E`, все зависят от `B`

### Карта и места (ключевой функционал)

- **F** — Интеграция 2GIS Map API
- **G** — Получение мест (API 2GIS)
- **H** — Поиск и фильтрация мест
- **I** — UI карты + отображение мест
- Зависимости: `F -> I`, `G -> H -> I`, все зависят от `B`

### Избранное

- **J** — Модель избранного + Repository
- **K** — Добавление/удаление избранного
- **L** — UI экрана избранного
- Зависимости: `J -> K -> L`, блок зависит от `G` (нужны места)

### Медиа мест

- **M** — Интеграция связи изображений и мест
- **N** — Отображение медиа
- Зависимости: `M -> N`, блок зависит от `G`

### Сообщество

- **O** — Модель постов/отзывов
- **P** — Загрузка постов
- **Q** — Создание поста
- **R** — UI ленты постов
- **S** — UI профиля
- Зависимости: `O -> P -> R`, `O -> Q`, `E -> S` (нужен пользователь)

### Инструменты

- **T** — Экстренные номера (UI + данные)
- **U** — Переводчик (offline)
- **V** — Конвертер валют
- Зависимости: `T`, `U`, `V` — независимы, выполняются после `B`

### Финал

- **W** — Интеграция всех модулей
- **X** — Тестирование
- **Y** — Сборка APK и деплой
- Зависимость: все блоки -> `W -> X -> Y`

## 2. Упрощённая структура PERT (логика зависимостей)

1. `A -> B -> {C -> D -> E}`
2. `{F -> I}`
3. `{G -> H -> I}`
4. `{J -> K -> L}`
5. `{M -> N}`
6. `{O -> P -> R}`
7. `{O -> Q}`
8. `{E -> S}`
9. `{T, U, V}`
10. `Все -> W -> X -> Y`

## 3. Примерные оценки времени (PERT)

| Задачи | Описание задач | Время выполнения |
| --- | --- | --- |
| A, B | Анализ и подготовка ТЗ | 14 д |
| C, D, E | Авторизация с помощью Firebase | 7 д |
| F, G, H | Интеграция карты на экран поиска (2GIS) | 15 д |
| P, Q, R | Реализация работы с постами и интерфейса для работы с постами | 10 д |
| J, K, L | Интерфейс избранного и работа со списком избранных мест | 7 д |
| T, U, V | Инструменты и работа с ними | 7 д |
| M, N | Внедрение медиа файлов, связанных с местом | 7 д |
| W, X, Y | Тестирование и подготовка релиза | 7 д |

## 4. Критический путь

Самая длинная цепочка (влияет на срок проекта):

`A -> B -> G -> H -> I -> W -> X -> Y`

Альтернативно:

`A -> B -> O -> P -> R -> W -> X -> Y`

Почему:

- карта + поиск — core функционал;
- посты — сложная логика + UI;
- интеграция объединяет все модули.
