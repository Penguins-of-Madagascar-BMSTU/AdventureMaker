# Техническое задание (ТЗ)

## 1. Общее описание проекта

Проект AdventureMaker представляет собой мобильное туристическое приложение. Его основная задача – сопроводить путешественника по Российским городам информацией об интересных или полезных местах. Также приложение предоставляет возможность общения и обмена впечатлениями с другими пользователями с помощью постов.

Цель проекта – создание туристического приложения, упрощающее путешествия по Российским городам, таким как Санкт-Петербург и Москва.

## 2. Описание артефактов и терминов (глоссарий)

- Clean Architecture – архитектура, состоящая из слоёв data, domain и presentation.
- Карта – данные о всех локациях города, загружаемые в виде тайлов с сервера 2GIS.
- Firebase Authentication – механизм аутентификации пользователя.
- Entity – сущность domain слоя, являющаяся частью бизнес-логики.
- DTO (Data Transfer Object) – объект передачи данных между клиентом и сервером.
- Repository – интерфейс доступа к данным.
- Use-Case – реализация бизнес-логики с помощью абстракции Repository.
- Retrofit2 – библиотека для обработки HTTPS-запросов.
- Place (Место) – туристическое место (достопримечательность, кафе, отель и т.д.).
- PlaceMedia – медиафайл (фото/видео), связанный с местом.
- Review (Отзыв) – отзыв пользователя о месте.
- Favorites (Избранное) – набор идентификаторов мест, добавленных в сохранённых пользователем.
- Currency – информация о курсе валют относительно выбранной валюты.

### Посты и отзывы

- Post (Фото-пост) – публикация пользователя с изображением и подписью.
- Перевод поста – перевод описания поста с русского на язык пользователя.


Пост является совокупностью фотографии и текста к ней, причём он привязан к id автора.

Отзыв является постом о каком-либо месте с оценкой от 1 до 5.

Место относится к одной из следующих категорий:

- банк;
- достопримечательность;
- музей (выставка);
- развлечение (мероприятие);
- отель (гостиница);
- кафе (ресторан).

Отзывы отображаются на экране с описанием места.

Уникальность поста или отзыва определяется его id.

### Пользователь

Сущность пользователя содержит только id, имя и email. Авторизация пользователя происходит по email и пароль: сначала проверяется наличие email и правильность пароля с помощью Firebase Authentication, а при успешной проверке пользователь с нужным email получается из Firebase realtime database. Таким образом, нет необходимости передавать пароли по сети и хранить их в сущности пользователя. Токена авторизации не предусмотрено.

### Перевод

Перевод осуществляется по специальной кнопке и работает без интернета с помощью библиотеки от google.

## 3. Продуктовое описание

Основные разделы:

1. Избранное.
2. Карта с поиском мест.
3. Сообщество: посты и профиль.
4. Инструменты: переводчик, конвертер валют и другие инструменты.

## 3.1 Избранное


На данном экране пользователь видит список мест, которые он добавил. Есть возможность отфильтровать их по категории.

UI-компоненты:

- карточка места с кнопкой удаления из избранного и подписью;
- прокручиваемый список карточек мест из 2 колонок;
- карточка категории: текст, обведённый овалом;
- прокручиваемый ряд из карточек категорий;
- карточка места.

## 3.2 Карта с поиском мест

Пользователь вводит строку для поиска места по названию и выбирает город и категорию. Категорию можно не выбирать. На карте отображаются найденные места.

Также на шторке можно посмотреть эти места списком: для каждого из них отобразится статус добавления в избранное.

UI-компоненты:

- карта (MapView);
- список с карточками категорий (FlowRow + Card);
- элемент списка мест c названием и кнопкой добавления в избранное;
- прокручиваемый список из мест, отображаемых на карте.
- строка поиска места;
- выпадающее меню с выбором города.

## 3.3 Сообщество: посты и профиль

Сначала пользователь видит экран авторизации.

После этого ему отображается список постов других пользователей.

Пользователь может перейти в профиль и посмотреть собственные посты и данные.

На экране с профилем также есть кнопка добавления поста. По нажатии появляется экран с созданием поста: есть возможность загрузить фото и написать текст. При создании поста можно добавить ссылку на место и оценку – тогда пост превращается в отзыв.

UI-компоненты:

- аватарка пользователя (Image);
- изображение поста (Image);
- текст поста (Text);
- оценка поста в виде иконок звёзд.

## 3.4 Инструменты

На экране с инструментами пользователь выбирает вид инструмента и в зависимости от выбора видит экран с использованием инструмента.

Список доступных инструментов:

1. Экстренные номера.
2. Конвертер валют.
3. Перевод.

## 3.5 Используемое API

Приложение получает все данные от публичных сервисов по HTTPS:

- запросы к Firebase для авторизации и получения текстовой информации.
- запросы к хранилищу изображений Timeweb Cloud;
- запросы к 2GIS для получения карты в виде тайлов;
- запросы к 2GIS для получения информации о местах;
- запросы к Flickr API для получения фотографий мест.

## 4. Техническое описание

## 4.1 Архитектура системы

Frontend:

- UI Layer – Activity/Composable функции.
- ViewModel Layer – управление состоянием UI (Android ViewModel / Jetpack Compose).
- слой бизнес-логики (UseCase).
- Repository Layer – слой обработки данных и взаимодействия с сетью.

## 4.2 Технологический стек

Технологии:

- Kotlin
- Android SDK
- Retrofit2
- Koin (Инъекция зависимостей)
- Coroutines
- Gson (JSON-сериализация)
- Firebase

Шаблоны проектирования:

- MVVM
- Clean Architecture (SOLID)

## 5. Условия эксплуатации

- Android 8.0 (API 26) и выше.
- Наличие интернет-соединения.

## 6. Правила выгрузки (деплой и обновление)

Развертывание и обновление:

- получение кода из GitHub;
- сборка подписанного релизного apk файла;
- выгрузка apk файла на устройство;
- запуск приложения.

Сборка apk производится с помощью android studio.

## 7. Минимальный функционал

## 7.1 Обязательные возможности

1. Просмотр карты выбранного города.

1. Редактирование пользователя в профиле.
2. Создание поста.
3. Просмотр постов пользователей.
